## ⚠️ 风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| Libvirt 权限问题 | 无法创建 VM | 使用 root 权限运行，配置 AppArmor |
| 并发 VM 操作 | 资源竞争、死锁 | 使用 Redis 分布式锁，队列化操作 |
| SSH 连接超时 | 部署失败 | 重试机制，超时配置，健康检查 |
| 磁盘空间不足 | VM 创建失败 | 监控磁盘使用，自动清理，扩容告警 |
| 网络 IP 耗尽 | 无法分配 IP | IP 池管理，NAT 配置，定期回收 |

## 📝 架构决策记录（ADR）

### ADR-001: 选择 Python/FastAPI 而非 Node.js/Express
- **决策**: 使用 Python/FastAPI 作为后端框架
- **理由**: Libvirt Python SDK 更成熟，AI 生态更丰富
- **权衡**: Node.js 异步性能略优，但 FastAPI 已足够
- **影响**: 团队需要 Python 开发经验

### ADR-002: 选择 PostgreSQL 而非 MySQL
- **决策**: 使用 PostgreSQL 作为主数据库
- **理由**: JSONB 支持更好，分区表功能强大
- **权衡**: MySQL 在简单查询上略快
- **影响**: 需要配置分区表

### ADR-003: 选择 KVM 而非 Docker
- **决策**: 使用 KVM 作为虚拟化方案
- **理由**: 完全隔离，安全性更高
- **权衡**: Docker 更轻量，但隔离性不足
- **影响**: 需要配置 Libvirt，运维复杂度增加

### ADR-004: 预充值计费模式
- **决策**: 采用预充值模式
- **理由**: 资金流好，风险可控
- **权衡**: 用户体验略差
- **影响**: 需要实现余额不足停服机制
