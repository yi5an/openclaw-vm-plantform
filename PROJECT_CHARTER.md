# OpenClaw VM Platform - Project Charter

## 项目概述
- **项目名称**: OpenClaw VM Platform
- **定位**: 虚拟机租赁平台，让用户快速拥有自己的 OpenClaw 实例
- **启动日期**: 2026-03-21
- **项目负责人**: @project

## 核心目标
1. 用户可以注册、登录、管理账户
2. 用户可以创建、启动、停止、删除虚拟机
3. 每个虚拟机预装 OpenClaw 环境
4. 实现按使用计费（时长/资源）
5. 提供实时监控和状态展示

## 范围定义

### IN（包含）
- 用户认证系统（注册/登录/权限）
- VM 全生命周期管理（CRUD + 启停）
- KVM/Libvirt 虚拟化集成
- 计费系统（按小时/资源）
- 监控面板（CPU/内存/磁盘/网络）
- 前端管理界面
- RESTful API

### OUT（不包含）
- 移动端 App（MVP 阶段）
- 多区域部署（初期单机房）
- 高级网络配置（VPC/负载均衡）
- 自动扩缩容

## 技术栈
- **前端**: React 18 + TypeScript + Vite
- **后端**: Python 3.11 + FastAPI
- **数据库**: PostgreSQL + Redis
- **虚拟化**: KVM + Libvirt
- **消息队列**: Redis（异步任务）

## 关键风险
1. **技术风险**: KVM/Libvirt API 复杂度 → 提前技术预研
2. **安全风险**: 多租户隔离 → 架构师重点设计
3. **性能风险**: 并发 VM 操作 → 引入队列异步处理
4. **依赖风险**: Libvirt 权限配置 → 需要宿主机 root 权限
